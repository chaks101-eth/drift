Direction II · Trip Bundle · Mobile + Desktop · Final spec v2

The whole trip, customized in one go.

The user taps "Reserve trip" → reviews the cart → walks through customizing flights (seat, baggage, meal) → customizing the hotel (room + add-ons like breakfast, late checkout, transfer, spa) → enters travelers → pays once → gets every PNR + reservation on one screen.

What changed in v2: the customization moments are now first-class steps. Step 3 covers flights (with seat maps + bags); step 4 covers the hotel (room pick + add-ons). The flow only feels like a "bundle" if users get to make the small decisions that matter — anything less is Drift guessing for them.

What we ship & in what order

Phase 1 (2–3 weeks): Steps 1, 2, 5, 6, 7 wired with Stripe Checkout backed by a manual concierge queue. Steps 3 (seats) and 4 (room + add-ons) ship as UI shells using mock data. Tests whether users will trust Drift with their card before any API contracts.

Phase 2 (4–6 weeks): Replace concierge with Duffel (flights + live seat maps) and Hotelbeds (hotels + live room inventory + add-on catalog). Steps 3 & 4 become real. Confirmation screen renders live PNRs and reservation IDs.

Phase 3 (post-launch): Activity provider integration (GetYourGuide / Viator), Apple/Google Wallet boarding passes, shareable confirmation links as a marketing surface.
