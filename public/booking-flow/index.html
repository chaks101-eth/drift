Three ways to own the booking.

Today every booking CTA on a Drift card redirects users to a third-party site. That breaks the trip experience and gives away the conversion. These are three product directions for keeping it in-app — from the cheapest MVP to the deepest moat. Pick a direction; then we build it.

I
Direction 01 · Concierge

Book it for me

User submits a request → Drift confirms via email in 5 minutes → user pays via Stripe link. No real-time inventory API needed. Ships in 2 weeks.

5 screens Stripe + email Manual queue scales to ~100/day

Direction 02 · Trip Bundle (recommended)

Book the whole trip in one checkout

One cart for all flights, hotels, activities. One payment. Unified itinerary with every confirmation number. The only path where Drift wins structurally vs Booking.com — they can't bundle inventory their teams don't co-own.

Recommendation

Build V1 first as the wedge, then evolve into V2 inside the same app surface. The concierge flow is shippable in ~2 weeks with zero API risk and tests whether users will actually trust Drift with a booking. Once you have ~500 paying bookings, the conversion data justifies the Duffel + Hotelbeds contracts and you upgrade to the trip-bundle flow — which is the long-term moat. V3 is the wrong direction unless we hear strong "I want seat selection in-app" feedback from real users; the premium UX doesn't pay for itself.
